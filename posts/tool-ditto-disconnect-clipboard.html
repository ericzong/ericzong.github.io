Ditto使用技巧：断开剪贴板

发表于 2021-11-17 更新于 2021-11-18 分类于 系统 ， 效率工具

默认设置下，在复制或剪切时，剪贴项会自动发送到 Ditto，因为 Ditto 默认连接到剪贴板。

如果不想复制/剪切的数据发送到 Ditto，可以“断开连接”，在托盘图标右键菜单中选择【断开剪贴板连接】菜单项即可。这样，数据就不会再发送到 Ditto。

想恢复连接，在托盘图标右键菜单中选择相同菜单项即可——当然此时菜单项的名称在断开后已转变为【连接到剪贴板】了。

为什么要断开连接？
呃……最重要的原因是涉及安全隐患。
Ditto 会记录复制的一切，包括用户名、密码等等敏感数据，并且以明文显示。存在信息泄露的风险！
虽然可以通过设置【选项-常规】中【保存复制项目的最大数量】和【粘贴条目的过期日期】两项从某种程度上减少风险，但不能规避。断开连接才是“一劳永逸”的。

断开连接有什么负面影响吗？
有。在另一篇文章《Ditto使用技巧：复制缓冲》中提及断开连接会使复制缓冲无法使用。

虽然我们解决了一个问题，但是断开连接后 Ditto 还有什么用？这就需要设置另外两个快捷键了。

在【选项-键盘快捷键】中设置【保存当前剪贴板内容】和【Copy And Save Current Clipboard】两个功能的快捷键。（通常笔者分别设置为 Win + Alt + X 和 Win + Alt + C。）前者用以将当前系统剪贴板中的内容保存到 Ditto，而后者是执行复制操作并将复制的内容保存到 Ditto。即我们依然可以将数据发送到 Ditto 中。

小结

  • 考虑到“安全风险”应该将 Ditto 设置为“断开剪贴板连接”状态
  • 为了在“断开”状态下可用，需要额外设置两个相关的“复制快捷键”。
