JSON Schema参考

模式关键字(Schema Keyword)

关键字 描述 有效值 用途
$schema 版本描述 字符串 指定 Schema 所使用的规范版本1
$id 唯一标识符 字符串 除用作唯一标识符，还能用于引用2
1. 较新的两个草案为 https://json-schema.org/draft/2020-12/schema 和 http://json-schema.org/draft-07/schema。 ↩
2. 可参考有关复用的相关章节。 ↩

模式注解关键字(Schema Annotation)3

关键字 描述 有效值 用途
title 标题 字符串 概要说明
description 描述信息 字符串 详细说明，通常较标题更长
3. 或称“元数据关键字”。 ↩

验证关键字(Validation Keyword)

基本类型

类型 描述
object 对象
array 数组
string 字符串
number 任意数字4
integer 整数5
boolean 布尔型
null 空
4. 可使用指数表示法。 ↩
5. 是否可带为 0 的小数部分，取决于具体实现。有些实现可能将 1.0 视为整数。 ↩

通用关键字

关键字 描述 有效值 验证
type 类型 string/string array，取值为基本类型名 取值为指定类型（之一）
enum 枚举 string array，至少有一个元素，且无重复值 取值存在于枚举中
const6 常量 任意 值与之相同
6. const 等价于仅有唯一值的 enum，是后者的语法糖（syntactic sugar）。 ↩

注意：type、enum、const 理论上可以同时使用，验证亦会同时生效，即为“交集”的效果。

对象相关

关键字 描述 有效值 验证
properties 属性 object，属性键值对 对应属性通过验证
required 必需属性 string array，至少有一个元素，且无重复值 object 实例必须有所有 required 定义的属性
additionalProperties7 其他属性 boolean/object 为 false 则禁止其他属性；为 object 时定义其他属性键值对
maxProperties 最大属性个数 integer object 实例的属性个数 ≤ maxProperties
minProperties 最小属性个数 integer object 实例的属性个数 ≥ minProperties
7. 其他属性指的是除 properties 定义之外的属性。 ↩

数组相关

关键字 描述 有效值 验证
items 定义元素 object/array 所有元素通过验证8
additionalItems 长度限制 boolean/object 为 false 时，不允许其他元素；为 object 时，其他元素通过验证 9
maxItems 长度限制 非负整数 数组长度 ≤ maxItems
minItems 长度限制 非负整数 数组长度 ≥ minItems
uniqueItems 唯一值 boolean 为 true 时，不能有重复值
8. 可配置为列表验证器(List validation)，对应为验证对象；或为元组验证器（Tuple validation），对应为验证对象数组。 ↩
9. 仅在 items 为数组时使用。 ↩

字符串相关

关键字 描述 有效性 验证
maxLength 最大长度 非负整数 字符串的长度必须≤该值
minLength 最小长度 非负整数 字符串的长度必须≥该值
pattern 正则表达式 字符串，有效的正则表达式10 字符串需匹配11正则表达式
10. 使用的是 JavaScript 的正则语法，即 ECMA 262。但目前可能并未完全，使用时需特别注意。 ↩
11. 仅要求正则表达式匹配字符串中部分即可，除非正则表达式为 ^...$ 的形式。 ↩

数字相关

关键字 描述 有效值 验证
multipleOf 整数倍 > 0 的数 取值需为指定数的整数倍
maximum 最大值 任意数 取值 ≤ maximum
exclusiveMaximum 最大值（不包含） 任意数 取值 < exclusiveMaximum
minimum 最小值 任意数 取值 ≥ minimum
exclusiveMinimum 最小值（不包含） 任意数 取值 > exclusiveMinimum
